# Forgemux Hub UX Review

**Date:** 2026-03-03  
**Scope:** Hub dashboard UI (`dashboard/`), static review of current implementation.

## Summary
The Hub dashboard’s IA is coherent and the visual system is consistent. The primary UX risks are around **action focus** (keyboard actions targeting the wrong decision), **session selection in Replay**, and **insufficient feedback** for certain actions. Secondary issues include weak separation of concerns in Attach, minimal connection telemetry, and non‑stateful replay tabs.

## Findings (Ordered by Impact)

1. **Decision hotkeys lack focus context**
   - **Issue:** Keyboard actions (`a/d/c`) always target the first decision in the filtered list.
   - **Impact:** High risk of acting on the wrong decision.
   - **Suggested fix:** Track “active decision” on hover/click and apply hotkeys to that ID.
   - **Location:** `dashboard/components/decisions.js`

2. **Replay has no session selector in‑view**
   - **Issue:** Replay relies on external selection (fleet/decisions). On refresh, it picks the first session with no UI to change it.
   - **Impact:** Users get “stuck” in a single session without a clear path to switch.
   - **Suggested fix:** Add a compact session list or dropdown in Replay header/sidebar.
   - **Location:** `dashboard/components/replay.js`, `dashboard/app.js`

3. **Attach view mixes “start” and “attach” without stronger separation**
   - **Issue:** The left pane blends “Start Session” controls with the sessions list.
   - **Impact:** Increases cognitive load; starting and attaching feel like one overloaded pane.
   - **Suggested fix:** Add a stronger divider, collapsible “Start Session”, or two‑column layout.
   - **Location:** `dashboard/components/attach.js`

4. **Connection status lacks detail**
   - **Issue:** Only shows `live/connecting/reconnecting`.
   - **Impact:** Operators can’t tell how stale the connection is or when it last updated.
   - **Suggested fix:** Show last update timestamp or “x seconds since update”.
   - **Location:** `dashboard/components/nav.js`

5. **Decision comments have no user feedback**
   - **Issue:** Comment action does not show success/failure feedback.
   - **Impact:** Unclear whether comment was submitted.
   - **Suggested fix:** Inline toast or temporary status badge near the card.
   - **Location:** `dashboard/app.js`, `dashboard/components/decisions.js`

6. **Repo identity degrades when workspace repos are empty**
   - **Issue:** Repo pills render as generic labels when workspace repo data isn’t populated.
   - **Impact:** Reduced clarity in Fleet view.
   - **Suggested fix:** Show fallback style or raw path as secondary text.
   - **Location:** `dashboard/components/fleet.js`, `dashboard/state.js`

7. **Replay tab state not persisted**
   - **Issue:** Replay tab resets to “diff” on refresh.
   - **Impact:** Mild annoyance for users monitoring terminal or logs.
   - **Suggested fix:** Store tab in hash params or localStorage.
   - **Location:** `dashboard/app.js`, `dashboard/components/replay.js`

8. **Empty states lack next‑step guidance**
   - **Issue:** Empty states state “no sessions/decisions” without guidance.
   - **Impact:** New users don’t know what to do next.
   - **Suggested fix:** Add a short hint, e.g., “Start a session in the Session tab.”
   - **Location:** `dashboard/components/fleet.js`, `dashboard/components/decisions.js`, `dashboard/components/attach.js`

## Strengths
- Consistent component language (cards, pills, badges).
- Good loading/error/empty state coverage.
- Decision cards are compact and readable; replay timeline is intuitive.
